## Learned User Preferences

- **Supabase:** The user has the Supabase MCP (`plugin-supabase-supabase`) connected. Prefer `list_projects`, `apply_migration`, and `execute_sql` for DDL/DML instead of asking the user to paste SQL in the dashboard—unless a task explicitly requires human review.
- **Local dev restarts:** When the user asks to start/restart this workspace, target the stock-screener app at `http://localhost:5174` and the backend at `http://localhost:3001`; verify the app returns HTTP 200 and avoid accepting Vite fallback ports that may be another project.

## Learned Workspace Facts

- Project conventions and stack notes for UI/planning work live in `frameworks.mdc` at the repo root (read it before assuming stack or workflow).
- Experts / “smart money” overlap: the UI uses routes `/experts` and `/experts/:slug` (`StockcircleExperts.tsx` and related detail pages). Backend combines StockCircle (`server/stockcircle/`, Supabase migrations such as `migration-stockcircle-experts.sql` and `migration-stockcircle-pct-of-portfolio.sql`, read-only `/api/stockcircle/...` in `server/http/registerStockcircleRoutes.js`), WhaleWisdom (`server/whalewisdom/`), and FMP Congress (`server/fmp/`, `FMP_API_KEY`, `docs/supabase/migration-fmp-congress.sql`); unified sync lives in `server/experts/runUnifiedExpertsSync.js` / `runExpertsSync.js`, exposed via cron-protected experts routes and `scripts/run-experts-sync.js` (shared `server/http/cronSecretAuth.js` pattern). Combined read model: `GET /api/experts/summary`. Manual StockCircle-only runs still use `scripts/run-stockcircle-sync.js`. Curated WhaleWisdom filer links not ingested by sync: `src/data/expertInvestorsWatchlist.js`. Some FMP institutional endpoints may be unavailable on lower subscription tiers (402 / plan errors).
- `/experts` is multi-source: `GET /api/experts/summary` (`server/experts/buildExpertsSummaryPayload.js`) **merges** the StockCircle overlap payload from `buildStockcircleSummaryPayload.js` (`popular` tickers via capped `v_stockcircle_ticker_popularity`, `expertWeightsByTicker` for the latest run) with **FMP Congress** gateway fields and recent `fmp_congress_trades`. Unified sync (`runUnifiedExpertsSync.js`) runs **WhaleWisdom** 13F + StockCircle + FMP together; WhaleWisdom positions are served separately (`/api/whalewisdom/...`, page `WhalewisdomFilerDetail.tsx` / `/whalewisdom-filers/:slug`), not folded into that summary matrix. The **consensus + “top performers”** UI still ranks experts using `buildSortedExpertUniverse` (`src/utils/expertConsensus.ts`) on the StockCircle-derived `popular` × positions slice—so row counts are often below total investors across all sources, and WhaleWisdom-only filers (e.g. default `situational-awareness-lp` in `server/whalewisdom/filerSlugs.js`) do not appear in that table unless they also show up in StockCircle overlap data.
- Expert overlap AI snapshot: `POST /api/experts/ai-insights` uses `resolveExpertsInsightsConfig()` in `server/llm/index.js`. **Precedence:** explicit `EXPERTS_INSIGHTS_PROVIDER` if set → else **Ollama** when `OLLAMA_API_KEY` is set (default model `minimax-m2.7:cloud`) → else **OpenRouter** when `OPENROUTER_API_KEY` is set (default `gemini-3-flash-preview:cloud`) → else Anthropic/OpenAI from dialogue env. Override model with `EXPERTS_INSIGHTS_MODEL`. `POST /api/cron/experts-sync` can allow same-origin browser triggers without a Bearer when `CRON_ALLOW_EXPERTS_BROWSER_SYNC=1` (`server/http/registerExpertsSyncRoutes.js`); production still requires secrets for other cron routes unless similarly opted in.
- Consensus tab money-flow narrative: `POST /api/experts/consensus-buys-ai` → `generateConsensusBuysInsights` uses the same resolver (Ollama-first when `OLLAMA_API_KEY` is set). `server/llm/index.js` normalizes assistant `content` (string or array parts), sets default OpenRouter `HTTP-Referer` / `X-Title` when env vars are unset, and supports `EXPERTS_AI_DEBUG=1`; consensus uses `reasoningFallback: false` on the first pass (hallucination retry may allow reasoning). Truncation mitigations include a higher default completion budget, optional extra completion rounds when `finish_reason` is `length` (`EXPERTS_CONSENSUS_BUYS_MAX_TOKENS` / `EXPERTS_CONSENSUS_BUYS_MAX_CONTINUATIONS` in `.env.example`), and stripping obvious lead-in lines such as `thought` / `Thinking:` before display. The `/experts` UI refreshes that request from a `summaryFingerprint` that does not require `latestRun` (needs `ok` + non-empty `popular`).
- On `/experts`, **main tabs** are **Consensus overlap**, **Overlap matrix** (experts×tickers grid when `popular` is non-empty), and **All experts · performance**; inside Consensus, vote-pattern groups are **nested sub-tabs** with per-group row counts. Consensus ticker tables (`ConsensusTable` in `StockcircleExperts.tsx`) default to sorting by the Score column (`conviction` key) descending.

- **AI Portfolio:** Route `/ai-portfolio` (`src/pages/AiPortfolio.tsx`); backend `server/aiPortfolio/` with routes registered in `server/http/registerAiPortfolioRoutes.js` (`GET /api/ai-portfolio/config`, `GET /api/ai-portfolio/summary`, `POST /api/ai-portfolio/simulate/daily`, `POST /api/ai-portfolio/simulate/daily-stream`, scheduler-related endpoints). - **Stock detail · options open interest:** Chart-adjacent call/put OI by strike uses `GET /api/options-open-interest/:ticker` with optional `expiration` / `quotes=1` query (`server/http/registerOptionsOpenInterestRoutes.js`, `server/optionsOpenInterest/service.js`); Supabase `options_open_interest_cache` is checked first, Yahoo is source of truth on cache miss/stale, and the cache TTL is 5 minutes. Fresh payloads read Supabase `bars_cache`, compute realized volatility from daily closes, scale by expiration DTE, and filter strikes to the `spot ± 2σ` band; if bars/volatility are unavailable, the service falls back to the older unfiltered OI payload. `StockDetail.tsx` renders `OptionsOpenInterestRail.tsx` beside the lightweight price chart; the price pane is `PRICE_CHART_HEIGHT = 600`, strike rows use the candlestick series `priceToCoordinate()` via `strikeCoordinates` so OI bars align vertically with the stock price y-axis, and the rail requests the full backend-filtered band instead of capping to nearest rows. The options-strategy UI (`OptionsStrategyVisualizer` / `src/utils/optionsStrategy.ts`) is the third desktop column with a **strategy dropdown** (e.g. bull put/bull credit spread, bear put spread, put credit spread, bear call spread, long call, short put / cash-secured put); payoff graph and summary metrics follow the selected strategy and strikes. Draggable `Sell Put` / `Buy Put` (and related) chips in `OptionsOpenInterestRail.tsx` use pointer capture, with **call-side grab styling green** and **put-side grab styling red**; default expiration selection prefers the nearest standard **monthly** expiration (`pickDefaultOpenInterestExpiration` in `src/utils/optionsOpenInterest.ts`), falling back to the first listed expiry when no monthly contract qualifies. Shared OI shaping helpers live in `src/utils/optionsOpenInterest.ts`. Options **gamma** remains a separate surface (`server/http/registerOptionsGammaRoutes.js` / `server/optionsGamma/service.js`).
